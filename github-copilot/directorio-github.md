# Empezando con el directorio `.github` en un proyecto

El directorio `.github` en un repositorio de GitHub es utilizado para almacenar configuraciones específicas del proyecto, como flujos de trabajo de GitHub Actions, plantillas de problemas y solicitudes de extracción, y otros archivos relacionados con la gestión del repositorio.

## Secciones más importantes del directorio `.github`

1. **Workflows**:  
   Contiene los flujos de trabajo de GitHub Actions, que automatizan tareas como pruebas, despliegues y otras integraciones. Los archivos de flujo de trabajo se almacenan en `.github/workflows` y están escritos en YAML.

2. **ISSUE_TEMPLATE**:  
   Plantillas para la creación de problemas (issues) en el repositorio. Estas plantillas ayudan a los colaboradores a proporcionar información estructurada.

3. **PULL_REQUEST_TEMPLATE.md**:  
   Una plantilla para solicitudes de extracción (pull requests), que asegura que los colaboradores incluyan detalles importantes sobre los cambios propuestos.

4. **CODEOWNERS**:  
   Define los responsables de revisar cambios en archivos o directorios específicos del repositorio.

5. **copilot-instructions.md**:  
   Un archivo opcional que proporciona instrucciones específicas para GitHub Copilot, ayudando a personalizar las sugerencias de código generadas.
